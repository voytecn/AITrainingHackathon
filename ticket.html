Description

Active production incident: PaymentBatch.aggregate is OOM-ing under load (heap at 87% just before crash), causing charges to succeed at Stripe but never commit to the local ledger. User u_4421 was charged $129.99 with no local transaction record — flagged for MANUAL RECONCILIATION.

Pattern has recurred 47 times in the last 30 minutes, so every occurrence is producing a new payment/ledger mismatch. Financial and customer-trust risk compounds with each minute the service stays up.

Steps to Reproduce

  1. Client calls POST /charge (PaymentController.charge:54) — e.g. req_id=7f3a9b, user_id=u_4421, amount 129.99 USD.
  2. PaymentService.handleCharge invokes the Stripe gateway — charge succeeds at Stripe.
  3. PaymentBatch.aggregate (PaymentBatch.java:142) attempts to build the batch — heap climbs past 87%.
  4. JVM throws java.lang.OutOfMemoryError before the local transaction is committed.
  5. Service returns 500 to client; Stripe charge stands, local ledger has no record.
  6. Reconciliation flag fires: “charged on stripe but local txn missing”.
